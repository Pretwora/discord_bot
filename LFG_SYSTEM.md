**Сервер:** Pretwora DS — Discord сервер WoW-игроков с проекта Sirus.su (реалм Neverest x3, WotLK 3.3.5a+)

## Что нужно реализовать — LFG система

### Два режима работы

#### 1. Запланированный рейд
- Модератор создаёт рейд командой `/рейд создать`
- Открывается Modal (форма Discord) с полями
- Бот постит красивый embed в канал `#рейды`
- Embed обновляется в реальном времени при каждой записи
- Есть дата/время, автонапоминания, закрытие набора

#### 2. Моментальный набор
- Команда `/рейд набор` — быстрая форма
- Минимум полей, быстрый постинг
- Пинг нужной роли (например `@Рейдер`)
- Упрощённый embed

## Детальная механика

### Запись в рейд (рейдеры)

**Кнопки в embed:**
- `✅ Записаться` — открывает Select Menu: выбор класса → выбор роли
- `❌ Отказаться` — убирает из списка
- `❓ Под вопросом` — записан но не уверен, лидер видит отдельно

**Классы WotLK (10 штук):**
- 🩸 Рыцарь Смерти (Death Knight) — танк или дд ближний
- 🐻 Друид (Druid) — танк, хил или дд
- 🏹 Охотник (Hunter) — дд дальний
- 🔥 Маг (Mage) — дд дальний
- ⚔️ Паладин (Paladin) — танк, хил или дд ближний
- ✨ Жрец (Priest) — хил или дд дальний
- 🗡️ Разбойник (Rogue) — дд ближний
- ⚡ Шаман (Shaman) — хил или дд
- 💀 Чернокнижник (Warlock) — дд дальний
- 🛡️ Воин (Warrior) — танк или дд ближний

**Роли:**
- 🛡️ Танк
- 💚 Хил
- ⚔️ ДД Ближний (melee)
- 🏹 ДД Дальний (ranged)

### Система очереди (КЛЮЧЕВАЯ ФИЧА)

- Каждая запись сохраняется с точным timestamp
- Порядок отображения в embed = порядок записи (кто первый — тот первый)
- Если слоты заполнены → игрок автоматически уходит в **резерв** (отдельный список под основным составом)
- Резервисты видят своё место в очереди: "Резерв #2 (Танк)"

**Сценарий "кто-то не пришёл":**
1. За 15 минут до рейда — бот пингует всех участников в embed
2. Лидер нажимает `[🚀 Начать рейд]` → бот переходит в режим отметки
3. Лидер отмечает каждого: `✅ Пришёл` / `❌ Не пришёл`
4. При отметке `❌ Не пришёл` — бот автоматически пишет в DM первому резервисту нужной роли:
   > *"🔔 Открылось место [Танка] в [ICC 25 HC] (15 июня, 20:00)! Ты следующий в очереди. Подтверди участие в течение **5 минут** кнопкой ниже."*
5. Кнопки в DM: `[✅ Иду!]` / `[❌ Не могу]`
6. Если не ответил за 5 минут → DM уходит следующему резервисту
7. Подтвердивший перемещается из резерва в основной состав в embed

### Система байеров (отдельный блок в embed)

**Что такое байер:** игрок который платит золото (внутри игры, вне бота) чтобы получить лут. Бот только **учитывает регистрацию** байеров, никакого золота через бота не проходит.

**3 категории лута (куска):**
- **ПРШ** — [расшифровка нужна от пользователя]
- **ВЖД** — [расшифровка нужна от пользователя]
- **ЛХМД** — [расшифровка нужна от пользователя]

**Количество слотов байеров зависит от типа рейда:**
- Каражан → 2 байера на каждый куска (итого 6 слотов)
- Гругг + Магик → 3 байера на каждый куска (итого 9 слотов)
- Другие рейды → настраивается вручную при создании

**Кнопка в embed:** `[🛒 Записаться как байер]`
→ открывает Select Menu с тремя опциями: ПРШ / ВЖД / ЛХМД
→ если слот занят — сообщение "Слот ПРШ заполнен, хочешь встать в резерв байеров?"

**Байеры тоже в очереди** — отдельная очередь для каждого типа куска.

## Шаблоны рейдов

Создаются один раз командой `/рейд шаблон создать`, переиспользуются вечно.

Хранят:
- Название (ICC 25 HC)
- Состав: сколько танков / хилов / дд ближних / дд дальних
- Байеры: включены или нет
- Тип байеров: какой рейд (Каражан / Гругг+Магик / кастомный)
- Кол-во кусков на каждый тип байеров

**Примеры стандартных шаблонов которые надо создать по умолчанию:**
- ICC 25 HC: 3 танка / 6 хилов / 8 ближних / 8 дальних, байеры отдельно
- ICC 10: 2 танка / 3 хила / 3 ближних / 2 дальних
- Каражан: байеры 2 куска на вид
- Гругг + Магик: байеры 3 куска на вид

## Напоминания (автоматические)

- **За 1 час** — бот пингует `@everyone` в канале рейда: "⏰ Через 1 час рейд [ICC 25 HC]! Состав: 20/25"
- **За 15 минут** — личный DM каждому участнику: "🔔 Через 15 минут собираемся! [ICC 25 HC] · 20:00"
- **После рейда** (через 30 минут после старта) — бот спрашивает лидера: "Отметить явку участников?"

## База данных — новые таблицы Prisma

```prisma
model RaidTemplate {
  id              String   @id @default(cuid())
  guildId         String
  name            String   // "ICC 25 HC"
  tanks           Int      // кол-во танков
  healers         Int
  meleeDps        Int
  rangedDps       Int
  buyersEnabled   Boolean  @default(false)
  buyerType       String?  // "karazhan" | "gruul" | "custom"
  buyerSlotsPerCut Int     @default(0)  // 2 или 3
  createdAt       DateTime @default(now())
  raids           Raid[]
}

model Raid {
  id              String        @id @default(cuid())
  guildId         String
  channelId       String        // канал где запостили
  messageId       String?       // ID сообщения embed (для редактирования)
  templateId      String?
  template        RaidTemplate? @relation(fields: [templateId], references: [id])
  name            String
  scheduledAt     DateTime?     // null = моментальный набор
  status          String        @default("open")  // "open"|"closed"|"started"|"finished"|"cancelled"
  leaderId        String        // discord ID лидера
  tanks           Int
  healers         Int
  meleeDps        Int
  rangedDps       Int
  buyersEnabled   Boolean       @default(false)
  buyerSlotsPerCut Int          @default(0)
  createdAt       DateTime      @default(now())
  signups         RaidSignup[]
  buyerSignups    RaidBuyerSignup[]
  remindersAt1h   Boolean       @default(false)
  remindersAt15m  Boolean       @default(false)
}

model RaidSignup {
  id          String   @id @default(cuid())
  raidId      String
  raid        Raid     @relation(fields: [raidId], references: [id], onDelete: Cascade)
  discordId   String
  username    String
  class       String   // "deathknight" | "druid" | "hunter" | ...
  role        String   // "tank" | "healer" | "melee" | "ranged"
  status      String   @default("pending")  // "confirmed"|"reserve"|"tentative"|"declined"|"noshow"
  queuePos    Int      // порядковый номер записи
  signedAt    DateTime @default(now())

  @@unique([raidId, discordId])
}

model RaidBuyerSignup {
  id          String   @id @default(cuid())
  raidId      String
  raid        Raid     @relation(fields: [raidId], references: [id], onDelete: Cascade)
  discordId   String
  username    String
  cutType     String   // "PRSh" | "VZhD" | "LKhMD"  (ПРШ | ВЖД | ЛХМД)
  status      String   @default("confirmed")  // "confirmed" | "reserve"
  queuePos    Int
  signedAt    DateTime @default(now())

  @@unique([raidId, discordId, cutType])
}
```

## Логика обновления embed

Embed обновляется (`message.edit()`) при каждом действии:
- Кто-то записался / отказался
- Кто-то перешёл из резерва в основной состав
- Лидер закрыл набор
- Рейд начался

Важно: messageId рейда хранится в БД — именно так бот находит нужное сообщение для редактирования.
